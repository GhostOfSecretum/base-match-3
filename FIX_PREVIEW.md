# Исправление проблемы с Preview

## Проблема
Игра не видна в Base Build Preview, но работает в реальном Base app.

## Возможные причины

1. **Preview использует другой способ загрузки SDK**
   - В preview SDK может быть доступен через другие глобальные объекты
   - Игра должна работать даже без SDK

2. **Проблема с загрузкой скрипта**
   - Скрипт может не загружаться в preview окружении
   - Проверьте консоль браузера в preview

3. **CORS или Content Security Policy**
   - Preview может иметь более строгие ограничения

## Решение

### 1. Проверьте консоль в Preview

1. Откройте Base Build Preview
2. Откройте DevTools (F12 или Command+Option+I)
3. Перейдите на вкладку "Console"
4. Проверьте, есть ли ошибки или логи

**Что искать:**
- `"=== SCRIPT.JS STARTING ==="` - скрипт загружается
- `"Test script loaded successfully!"` - тестовый скрипт работает
- Ошибки (красным цветом) - укажут на проблему

### 2. Проверьте Network

1. В DevTools перейдите на вкладку "Network"
2. Обновите страницу preview
3. Найдите `script.js` в списке
4. Проверьте статус:
   - **200** - файл загружается успешно
   - **404** - файл не найден
   - **CORS error** - проблема с доступом

### 3. Если скрипт не загружается

Проблема может быть в том, что preview блокирует загрузку скрипта. В этом случае:
- Игра все равно должна работать в реальном Base app (что и происходит)
- Preview может иметь ограничения, которые не влияют на реальное приложение

### 4. Временное решение

Если preview не критичен для тестирования:
- Игра работает в реальном Base app ✅
- Это главное - пользователи будут использовать реальное приложение
- Preview - это инструмент разработки, не финальная версия

## Проверка в реальном Base app

1. Откройте Base app
2. Найдите ваш пост с приложением
3. Нажмите "Play Now" или откройте приложение
4. Игра должна загрузиться и работать

Если игра работает в реальном Base app - все в порядке! Preview может иметь ограничения, которые не влияют на реальное использование.
